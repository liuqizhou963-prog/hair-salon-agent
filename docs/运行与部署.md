# 运行与部署

## 本地演示

```powershell
pip install -r requirements.txt
Copy-Item "配置模板-会上传-只填占位符.env.example" .env
python -m backend.database.init_db
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

打开客户端 `http://localhost:8000/`，员工端 `http://localhost:8000/staff`，接口文档 `http://localhost:8000/docs`。

## 数据库

- 开发演示默认使用根目录 `hair_salon.db`。
- 测试由 `tests/conftest.py` 强制使用独立内存 SQLite，不会污染演示库。
- 正式环境设置 `DATABASE_URL=postgresql+psycopg2://用户名:密码@主机:5432/数据库名`。
- 正式环境先执行 `alembic upgrade head`，当前迁移头为 `0005_add_member_expiry`。

### PostgreSQL 部署检查

部署环境需要分别验证“端口可达”和“账号能登录”，前者通过并不代表后者已经配置完成：

```powershell
Test-NetConnection -ComputerName 127.0.0.1 -Port 5432
python -c "import psycopg2; print(psycopg2.__version__)"
$env:DATABASE_URL = "postgresql+psycopg2://用户名:密码@主机:5432/数据库名"
alembic upgrade head
```

本机最终验收时 5432 端口可达、`psycopg2` 已安装，但 `.env` 没有配置 `DATABASE_URL`，并且没有可用的 PostgreSQL 密码，因此应用按设计使用 SQLite。正式部署前必须由部署环境提供真实连接串，再执行迁移和应用启动检查；不能把“端口可达”当成数据库部署完成。

## 模型与 RAG

没有模型 Key 时，客户侧规则 Agent 和员工侧查询 Graph 仍可运行；护理知识使用本地 BM25。配置 OpenAI-compatible API 或 Ollama 后，才启用 LangChain 模型调用；配置 `RAG_USE_CHROMA=true` 并准备 embedding 后，才启用 Chroma。

## Docker Compose 部署

Docker 方案使用 PostgreSQL 保存正式数据，应用容器启动时自动执行 `alembic upgrade head`，迁移成功后再启动 Uvicorn。生产密钥不写进仓库，启动前在终端设置：

```powershell
$env:AUTH_SECRET_KEY = "请替换为至少 32 位随机字符串"
docker compose up --build
```

启动后访问 `http://localhost:8000/`、`http://localhost:8000/staff` 和 `http://localhost:8000/docs`。停止容器但保留数据库卷使用 `docker compose down`；需要清空演示数据库时才使用 `docker compose down -v`。

Compose 中的 `RATE_LIMIT_ENABLED=true` 表示生产容器会启用 API 限流；当前限流器是单进程内存实现，扩展到多个应用副本前需要替换成 Redis 等共享计数器。
