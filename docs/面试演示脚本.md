# 3 分钟面试演示脚本

## 开场

这是一个美发门店智能运营系统。客户侧可以登录、预约、查看余额和通知；员工侧可以查看全店预约、客户会员和留存任务；AI 负责查询和运营辅助，但关键写操作必须人工确认。

## 演示顺序

1. 客户注册并登录 H5，说明 JWT 保存在客户端，请求通过 `Authorization` 传给后端。
2. 查询发型师和时间槽，创建一次护理预约，说明预约事实由数据库保存，不是前端数组。
3. 切到 `/staff`，用员工账号登录，查看今日预约和客户会员。
4. 在员工助手输入“今天有哪些预约”，展示回答中的 `tool:get_salon_schedule` 和 `database:staff_schedule` 来源。
5. 输入“染发后怎么护理”，展示 RAG 来源，说明实时业务数据不放进知识库。
6. 打开留存提醒，运行运营分析，展示流失风险、余额客户和会员到期建议。
7. 在预约调整区输入预约 ID 和新时间槽 ID，生成调整方案。此时数据库不变。
8. 点击拒绝，说明任务结束但业务不变；重新生成后点击确认，展示预约变更、客户通知和审计记录。
9. 回到客户端刷新通知，证明 AI 建议经过人工确认后才形成真实业务结果。

## 面试收尾

项目的重点不是让大模型直接操作数据库，而是把模型能力放在可控的工具和流程中。LangChain 提供模型和工具适配，LangGraph 负责节点编排，RAG 负责护理知识检索，SQLAlchemy 负责实时业务数据和事务。
