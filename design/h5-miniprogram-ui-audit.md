# H5 与小程序页面差异审核报告

审核时间：2026-07-10  
审核对象：
- H5：`frontend/index.html`，运行地址 `http://127.0.0.1:8787/index.html`
- 小程序：`frontend/miniprogram`

## 审核结论

当前小程序不是 H5 的完整等价迁移，而是一个简化版小程序。底部主导航已经对齐为「预约 / 我的」两项，但页面结构、功能入口、视觉规格、组件细节和交互状态仍有大量差异。

最关键的问题有 4 类：

1. H5「我的」页功能入口远多于小程序，小程序缺少充值、到店记录、积分详情、个人资料、推送通知、隐私政策等页面。
2. H5「预约」页是双列老师卡片，小程序已改成单列，视觉密度和 H5 不一致。
3. H5 有统一的手机壳式页面框架、顶部状态栏、返回按钮、底部 icon 导航；小程序使用原生导航栏和原生 tabBar，视觉结构不一致。
4. H5 的 hero、卡片、图标、按钮、空状态、加载状态、选中状态比小程序完整，小程序目前缺少多个状态和细节。

| 我的 | 有 | 有 | 结构差异大 |

| 老师排档 | 有 | 有 | 基本功能有，缺少空状态和细节 |
| 余额退款 | 有 | 有 | 文案、输入约束、布局细节不同 |
| 充值 | 有 | 无 | 必须补齐 |

| 我的积分 | 有 | 无 | 必须补齐 |
| 到店记录 | 有 | 无 | 必须补齐 |

| 推送通知 | 有 | 无 | 必须补齐 |

证据：
- H5 页面注册包含 `mine/recharge/refund/rechargeOk/points/history/profile/push/privacy/booking/teacherSchedule`。
- 小程序 `app.json` 只注册 `mine/booking/teacher-schedule/refund`。

## 全局框架差异

### 1. 页面外壳

H5：
- 页面最大宽度是 430px。
- 有模拟手机容器 `.app`。
- 背景有浅蓝和荧光黄径向光感。
- 有自定义状态栏、顶部栏、返回按钮、底部导航。

小程序：
- 使用原生小程序页面，没有 430px 手机容器。
- 使用原生 navigationBar。
- 使用原生 tabBar。
- 没有 H5 的自定义返回按钮和顶部标题布局。

影响：
- 小程序视觉上更像原生应用，H5 更像高保真手机样机。
- 如果目标是“和 H5 一模一样”，需要在小程序中模拟 H5 顶部栏和视觉容器；如果目标是“小程序自然体验”，原生导航可接受，但需要统一视觉密度。

### 2. 底部导航

H5：
- 两项：「预约 / 我的」。
- 每项有 SVG 图标和文字。
- 固定高度 72px。
- 顶部有边框和阴影。

小程序：
- 两项：「预约 / 我的」。
- 目前没有图标。
- 使用原生 tabBar，无法完全复刻 H5 的 SVG icon 样式，除非配置 tabBar iconPath。

建议：
- 若追求一致，应补充 tabBar 图标资源。
- 若接受原生样式，当前两项数量已经正确。

## 「我的」页差异

### H5 结构

H5「我的」页包含：
- 钱包 hero：大号余额 `￥328.00`
- 用户信息：姓名、GOLD 会员、积分
- 四个快捷入口：充值、我的预约、到店记录、我的积分
- 指标卡：可用余额、可用积分
- 我的预约列表/空状态
- 设置卡片：个人资料、推送通知、隐私政策

### 小程序结构

小程序「我的」页包含：
- 通用 hero：标题「我的」
- 用户信息一行：姓名、会员等级、积分
- 指标卡：可用余额、可用积分
- 我的预约列表/空状态
- 没有钱包大余额区
- 没有四个快捷入口
- 没有个人资料、推送通知、隐私政策

### 主要差异

| 细节 | H5 | 小程序 | 影响 |
|---|---|---|---|
| 首屏主视觉 | 钱包余额 hero | 普通蓝色 hero | 首屏重点完全不同 |
| 余额展示 | `￥328.00` 大号展示 | 指标卡里 `￥328` | 金额层级不一致 |
| 快捷入口 | 4 个 icon 按钮 | 无 | 功能缺失 |
| 设置列表 | 3 项 line-item | 无 | 功能缺失 |
| 空预约状态 | 卡片内标题、说明、按钮 | 有类似内容 | 基本一致 |
| 预约卡片 | 三列：头像、信息、状态 pill | 小程序两列，状态换到第二行 | 布局不一致 |

必须补齐：
- 充值入口和充值页。
- 到店记录入口和页面。
- 我的积分入口和页面。
- 个人资料、推送通知、隐私政策入口和页面。
- 钱包 hero 的大余额展示。

## 「预约」页差异

### H5 结构

H5「预约」页包含：
- hero 标题：`预约` + `发型师` 分行
- hero 文案：选择常用老师后，可以只看他的排档，快速找到熟悉的剪发时间。
- hero 操作按钮：只看常用老师 / 我的预约
- 选择美发老师 section
- 老师卡片双列网格
- 老师卡片有图片、认证图标、简介、预约数、作品数、常用按钮
- 选中常用老师后卡片有 selected 状态
- 可切换“只看常用老师 / 查看全部老师”

### 小程序结构

小程序「预约」页包含：
- hero 标题：预约发型师，单行
- hero 文案：先选择常用老师，进入老师排档后再选择可约时间。
- 没有 hero 操作按钮
- 老师卡片单列列表
- 老师卡片有图片、认证符号、简介、预约数、作品数、常用按钮
- 没有 selected 状态
- 没有“只看常用老师”筛选

### 主要差异

| 细节 | H5 | 小程序 | 影响 |
|---|---|---|---|
| 标题排版 | 两行大标题 | 单行标题 | 视觉不一致 |
| hero 操作 | 2 个按钮 | 无 | 功能缺失 |
| 老师列表 | 双列卡片 | 单列横向卡片 | 密度和视觉完全不同 |
| 图片比例 | 接近方形 | 固定 184rpx 横向缩略图 | 图片观感不同 |
| 简介行数 | 2 行 | 3 行 | 卡片高度不同 |
| 统计展示 | 图标 + 数字 | 文本：312 约、48 作品 | 图标缺失 |
| 常用状态 | 已常用/设为常用，含 active 样式 | 始终显示设为常用 + | 状态缺失 |
| 筛选 | 只看常用老师 | 无 | 功能缺失 |

必须决策：
- 如果小程序要严格对齐 H5，应恢复双列老师卡片，并解决之前遮挡问题。
- 如果小程序保留单列，则需要在设计文档中明确这是平台适配差异，不再说它与 H5 一致。

## 「老师排档」页差异

### H5 结构

H5 在未选择老师时：
- 显示空状态：还没有选择老师
- 按钮：返回选择老师

H5 在选择老师后：
- hero 标题：老师 / 排档分行
- hero 文案：老师名 + 专长
- 老师信息卡：头像、姓名、经验、评分、常用老师 pill
- 可约时间 section
- 时间段按日期分组
- 每个日期显示“几个时间段可选”
- 时间卡双列

### 小程序结构

小程序：
- 进入页面依赖 `id` 或 `selectedStylistId`
- 未选择老师时没有专门空状态保护
- 老师信息卡有头像、姓名、经验、评分、常用老师 pill
- 可约时间有日期分组和双列时间卡
- section 右侧多了“点时间段预约”

### 主要差异

| 细节 | H5 | 小程序 | 影响 |
|---|---|---|---|
| 未选老师状态 | 有完整空状态 | 无显式空状态 | 可能渲染空数据或报错风险 |
| 老师信息卡 | 三列：头像、信息、pill | 两列，pill 第二行 | 布局不一致 |
| 时间卡高度 | 92px | 136rpx | 视觉高度不同 |
| 时间卡圆角 | 16px | 22rpx | 视觉规格不同 |
| section 说明 | H5 section 只显示标题 | 小程序右侧显示点时间段预约 | 文案差异 |

建议：
- 补未选择老师空状态。
- 对齐老师信息卡结构。
- 统一时间卡高度、圆角、间距。

## 「余额退款」页差异

### H5 结构

H5：
- hero 标题：余额 / 退款分行
- 文案：可将会员余额退回原支付账户，提交后由门店确认处理。
- 指标卡：可退余额、预计到账
- 退款卡片：退款金额、说明、数字输入框、申请退款按钮
- input 有 `min`、`max` 和 `aria-label`
- 说明文案为“默认退回全部可用余额，也可以到店后由店员调整。”

### 小程序结构

小程序：
- hero 标题：余额退款，单行
- 指标卡同样存在
- 退款卡片同样存在
- input 是 number，但没有 min/max
- 说明文案为“默认退回全部可用余额，也可以由店员调整。”

### 主要差异

| 细节 | H5 | 小程序 | 影响 |
|---|---|---|---|
| 标题 | 分行 | 单行 | 视觉不一致 |
| 说明文案 | 有“到店后” | 缺少“到店后” | 文案不一致 |
| 输入约束 | min/max | JS 校验但 WXML 无 min/max | 体验不一致 |
| 输入辅助 | aria-label | 无等价 accessibility 属性 | 可访问性不足 |

建议：
- 小程序 input 加 `min="1"`，并使用 `placeholder` 或明确金额单位。
- 文案与 H5 完全一致。

## 数据内容差异

### 老师简介

H5 老师简介更长，例如：
- “擅长根据脸型做轻盈短发与空气感刘海，风格干净、自然、好打理。”

小程序简介更短，例如：
- “轻盈短发与空气感刘海，风格自然好打理。”

影响：
- 即使布局一致，卡片内容长度也会不同。
- H5 用 2 行截断，小程序用 3 行截断，会让卡片高度进一步不同。

建议：
- 统一 `bio` 文案源。
- 统一截断行数。

### 预约数据

H5：
- `demoAppointments` 和 API appointments 合并。
- 预约状态支持 confirmed/pending/completed/cancelled。

小程序：
- `appointments` 初始为空。
- 预约成功后只插入 confirmed。

影响：
- H5 可展示更多状态，小程序只有单一状态。
- 我的预约列表视觉和状态覆盖不完整。

## 组件规格差异

| 组件 | H5 | 小程序 | 建议 |
|---|---|---|---|
| Hero | 有 hair-logo、伪元素发丝装饰、标题分行 | 无 logo，标题单行 | 若追求一致，补 logo 和分行 |
| 底部导航 | icon + label | label only | 补 tabBar icon |
| 老师卡片 | 双列 profile card | 单列横向 card | 选定一种并统一 |
| 认证标识 | SVG check | 文本 `✓` | 改成 image/icon 或统一符号 |
| 统计 | SVG icon + 数字 | 中文文字 | 补图标或 H5 也改文字 |
| 按钮 | H5 有 active/disabled/pressed | 小程序状态少 | 补 disabled/active 样式 |
| 空状态 | 有 SVG icon | 小程序无 icon | 补统一空状态组件 |
| 输入框 | 有 min/max/placeholder/aria | 基础 input | 补输入约束和提示 |

## 交互差异

| 交互 | H5 | 小程序 | 结论 |
|---|---|---|---|
| 只看常用老师 | 有 | 无 | 必须补齐 |
| 设为常用 | 设置后进入排档，且状态可持久化 localStorage | 点击即进入排档，未体现已常用 | 语义不一致 |
| 选择老师 | 点卡片进入排档 | 点卡片进入排档 | 基本一致 |
| 预约成功 | alert 后回我的 | toast 后回我的 | 平台差异可接受 |
| 退款成功 | alert 后回我的 | toast 后回我的 | 平台差异可接受 |
| 返回 | H5 自定义 back stack | 小程序原生返回 | 平台差异可接受 |
| 加载态 | H5 有正在加载老师/排档/预约 | 小程序无加载态 | 建议补齐 |

## 优先级整改清单

### P0：必须先决定的设计口径

1. 小程序是否要“像素级复刻 H5”。
2. 老师列表到底用 H5 的双列卡片，还是保留小程序单列卡片。
3. 小程序是否保留原生 navigationBar/tabBar，还是模拟 H5 顶部和底部。

### P1：必须补齐的功能差异

1. 补充值页。
2. 补充值成功页。
3. 补我的积分页。
4. 补到店记录页。
5. 补个人资料页。
6. 补推送通知页。
7. 补隐私政策页。
8. 补“只看常用老师 / 查看全部老师”筛选。
9. 补常用老师 selected/active 状态。

### P2：必须统一的视觉差异

1. Hero：标题分行、logo、装饰、文案。
2. 老师卡片：列数、图片比例、圆角、阴影、统计图标、简介行数。
3. 预约卡片：状态 pill 的位置。
4. 时间卡：高度、圆角、内边距、字体大小。
5. 指标卡：圆角、内边距、金额字号。
6. 空状态：图标、说明、按钮。

### P3：体验和可访问性

1. tabBar 补图标。
2. 输入框补 min/max/placeholder。
3. 图片补可访问性描述。
4. 按钮补 disabled 状态。
5. 加载态和空态补齐。

## 建议落地路线

推荐先按 H5 作为主设计源同步小程序：

1. 先补页面清单，不急着调像素。
2. 再抽小程序公共组件：hero、metric-card、empty-card、appointment-card、stylist-card、slot-card。
3. 再决定老师列表是双列还是单列。
4. 最后用微信开发者工具逐页截图对照 H5。

如果你希望“小程序更像原生小程序”，则应把 H5 也改成类似小程序的单列卡片和原生感布局，否则两边永远会看起来不一样。
